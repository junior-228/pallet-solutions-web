# Pallet Solutions USA - Website (Next.js rebuild)

An in-progress rebuild of the Pallet Solutions USA marketing and customer-tools
site, built on the Next.js App Router. It replaces an older site with a modern,
component-driven front end and a build-time data pipeline.

> Status: active rebuild, not yet the production deployment.
> The current live site is https://palletsolutionsusa.com.

## What it is

Pallet Solutions USA is a managed pallet sourcing company. This site is the
public front door and the self-serve tooling around it:

- Vendor Network map - an interactive map of pallet vendors (Leaflet +
  react-simple-maps) with search and coverage views.
- Market intelligence - a published cost index and regional market "reads"
  rendered from a build-time data pipeline over public federal data.
- Market Pulse - a single-region cost snapshot.
- Lead capture / RFQ - quote-request and contact flows that post to
  serverless endpoints.

## Tech stack

- Framework: Next.js 16 (App Router, static export)
- Language: TypeScript
- UI: React 19, Tailwind CSS 4
- Maps / geo: Leaflet, react-simple-maps
- Data: Supabase + Airtable via serverless functions and a build-time export
- Hosting: Netlify

## Architecture notes

- App Router with a static export - pages are pre-rendered; dynamic data is
  baked at build time or fetched from serverless endpoints.
- A single source of truth for published values (JSON in `data/`, generated by
  `scripts/`) so the same number renders on every surface.
- Secrets are read from environment variables only (`process.env.*`); none are
  committed.

## Getting started

Prerequisites: Node.js 20+ and npm.

```bash
npm install
npm run dev      # dev server at http://localhost:3000
npm run build    # production build / static export
```

### Environment variables

Create `.env.local` in the project root and supply your own values (nothing
below is real):

```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
AIRTABLE_API_KEY=your-airtable-pat
AIRTABLE_BASE_ID=your-base-id
RESEND_API_KEY=your-resend-key
```

`.env*` files are gitignored and must never be committed.

## Project structure

```
app/          # App Router routes (home, market-pulse, atlas, vendors, ...)
components/   # React components (vendor finder, regional reports, maps, ...)
lib/          # data helpers (cost index, lead submission, Supabase client)
scripts/      # build-time data pipeline
data/         # generated JSON read at build time
public/       # static assets
```
